## Part 4 — Recommendation for Phase 2

Because every `\includegraphics` call already carries an `alt={…}` option, **the cheapest and safest Phase 2 plan is to leave the call sites untouched** and simply re-enable PDF tagging once the TeX Live 2025 tagpdf/amsmath conflict is resolved. If the team wants assistive-tech to surface the alt text **today** (before tagging is re-enabled), the recommendation is to add the tooltip wrapper below and rewrite each call.

### Recommended macro (only if Phase 2 chooses to rewrite call sites)

Add a new file `book/preamble/preamble-accessibility.tex` containing:

```latex
% book/preamble/preamble-accessibility.tex
\usepackage{pdfcomment}   % provides \pdftooltip{visible}{tooltip}
% \altincludegraphics[opts]{path}{alt-text}
\newcommand{\altincludegraphics}[3][]{%
  \pdftooltip{\includegraphics[#1]{#2}}{#3}%
}
```

Then `\input{preamble/preamble-accessibility}` from `book1-main.tex` immediately before `\begin{document}` (after `packages-and-commands` has loaded `graphicx` and `hyperref`).

Caller pattern (3 args, options optional):

```latex
\altincludegraphics[width=.4\textwidth]{figures/LyryxLogo.eps}{Lyryx Learning logo: stylized green checkmark over the lowercase letter L.}
```

Why `\pdftooltip` from `pdfcomment`:

- `pdfcomment` is **not currently loaded** by the project — verified across `packages-and-commands.tex` and every preamble file. Loading it adds the package plus its standard prereqs (`zref-abspage`, `pdfbase`), all in standard TeX Live.
- The tooltip is rendered as a hover popup in every mainstream PDF viewer, exposing the alt text to sighted readers and to most assistive-tech tooling **without requiring the tagging pipeline**.
- It works whether or not `\DocumentMetadata{tagging=on}` is uncommented later.
- The 3-arg signature keeps the merger's edit trivially mechanical: extract the current `alt={…}` value from the options, drop the `alt=` key from the options, and rewrite `\includegraphics[OPT]{PATH}` as `\altincludegraphics[OPT]{PATH}{ALT}`. Existing `width=…`/`scale=…` options are preserved.

### Alternative (zero new dependencies)

If Phase 2 wants no new package dependency, keep the calls as-is. The `alt={…}` key already does the right thing **once `\DocumentMetadata{tagging=on}` is reactivated**. Phase 2 then becomes a no-op for `.tex` edits; the only change is to fix the tagpdf/amsmath conflict that forced the metadata block off.

If Phase 2 still wants a thin wrapper for future-proofing without `pdfcomment`:

```latex
\newcommand{\altincludegraphics}[3][]{%
  \includegraphics[#1,alt={#3}]{#2}%
}
```

This is identical in behavior to what each call already does; it just centralizes the option name.
